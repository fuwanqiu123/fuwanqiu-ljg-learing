# UX 设计流程：共情、定义与构思 · 闪卡问题

> 以下问题基于课程核心知识设计，仅列问题，不附答案。复习时请主动检索，不要偷看笔记。

---

1. 在 UX 设计语境下，"同理心"(empathy) 与 "同情"(sympathy) 的根本分水岭在于研究者对用户的情感状态采取了怎样不同的心理动作？为什么这种差异会直接影响最终设计问题的准确性？

2. 当 UX 研究者招募访谈参与者时，为什么要刻意追求 "代表性样本"(representative sample)，尤其是在纳入那些 "过往研究中被忽视的用户群体" 方面？忽视这一点可能导致设计出现什么系统性偏差？

3. 在招募阶段，"筛选问卷"(screener survey) 的核心功能是什么？它与后续制定的 "访谈目标" 之间存在怎样的逻辑依赖关系？

4. 一封能有效提升陌生人参与意愿的访谈邀约邮件，除了说明项目背景和访谈时间地点外，还应包含哪三个关键要素？请从降低用户决策成本和建立信任的角度解释它们的作用。

5. 在访谈正式开始前的 "破冰阶段"，研究者明确告知参与者 "这次访谈没有标准答案"，这句话的核心设计意图是什么？它试图消解用户哪一类常见的心理防御机制？

6. 与封闭式问题（例如 "你喜欢这个功能吗？"）相比，以 "what" 或 "why" 开头的开放式访谈问题在挖掘用户真实体验方面具有什么结构性优势？请从 "信息量" 和 "意外发现" 两个维度说明。

7. 在用户访谈过程中，研究者为什么要同步记录用户的 "原话引用" 和 "非语言行为观察"（例如表情、肢体动作、语速变化）？如果只记录其中一类，会在后续制作同理心地图时造成什么信息缺口？

8. 同理心地图 (Empathy Map) 中，专门用于收录用户 "未说出口的心理活动" 的象限是哪个？研究者通常需要通过哪些非直接证据来推断并填充这一象限？

9. 当研究者观察到用户在访谈中频繁皱眉、叹气或出现犹豫性沉默时，这类非语言信息应当被记录在同理心地图的哪个象限？为什么它不能被简单视为 "用户说了什么" 的附属品或同一类信息？

10. 在将多个用户的访谈反馈整合为一张 "聚合版" 同理心地图时，研究者应当遵循什么分组原则来决定哪些用户的反馈可以被合并到同一张图中？在什么情况下必须拆分为多张独立的同理心地图？

11. UX 设计中的 "痛点" (pain points) 被划分为四大类型。当用户为了完成一个目标而在多个操作步骤之间反复受阻、无法顺畅地从 A 点到达 B 点时，这类体验问题属于哪一类痛点？它与 "产品本身质量缺陷" 所导致的痛点在解决思路上有何本质不同？

12. 2007 年苹果推出智能手机之前，为什么市场上几乎没有消费者主动提出 "我需要一部智能手机"？这个案例揭示了 UX 研究者在理解 "用户已知需求" 与 "潜在需求" 之间存在怎样的认知陷阱？研究者应如何规避这一陷阱？

13. 与单纯展示统计数据（例如 "目标用户群体有 50 万人"）相比，为设计团队创建具体的 "用户画像" (Persona) 能在沟通和产品决策中产生什么独特的心理效应？请从 "团队共情" 和 "方案论证" 两个角度分析。

14. 一个成熟的 UX 项目为什么需要为不同的核心用户群体分别建立独立的用户画像，而不是用一个 "平均用户画像" 来代表所有人？当设计师用多个画像来交叉检验同一设计方案时，本质上是在测试产品的什么属性？

15. 在构建用户画像时，为什么要为虚构用户设计一句 "口头禅" (catchphrase) 或一段带有细节的个人背景故事？这类看似与功能无关的 "软性信息"，在说服利益相关者 (stakeholders) 时发挥了什么关键作用？

16. 用户访谈收尾时，很多经验丰富的研究者会在最后问一句 "你还有什么想补充的吗？" 为什么这个看似 "闲聊式" 的开放问题往往能在访谈结束时产生最高密度的洞察？它触发了用户的什么心理机制？

17. 在正式访谈前，研究者为什么要提前把访谈脚本大声朗读并练习一遍？如果跳过这一步，在实际访谈现场最可能出现哪类会严重损害信息质量的问题？
