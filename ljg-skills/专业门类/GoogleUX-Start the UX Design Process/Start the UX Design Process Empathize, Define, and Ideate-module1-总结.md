# Google UX 课程 · 模块一：与用户产生同理心（Empathize）

> **核心要义**：UX 设计不是解决"你以为"的问题，而是解决"用户真实感受到"的问题。同理心不是天赋，是一套可以练习的方法论。

---

## 一、同理心到底是什么？

先区分三个容易混的词：

| 词 | 本质 | UX 中的问题 |
|---|---|---|
| **怜悯 Pity** | "你好惨"，居高临下 | 把用户当弱者，设计带着施舍感 |
| **同情 Sympathy** | "我理解你"，但站在岸上 | 知道痛点，但没真正进入对方的处境 |
| **同理心 Empathy** | "我就是你"，共享心理与情感体验 | ✅ 能预判用户在想什么、会怎么反应、情绪如何起伏 |

**一句话记住**：UX 设计师要当用户的"临时分身"，不是当用户的"慈善家"。

---

## 二、用户访谈：从"找人"到"聊完"的完整链路

### 1. 招募参与者

**先定目标**：别急着抓人，先问自己——
- 我想通过这次访谈**学到什么**？
- 哪些用户痛点我必须亲自感受？
- 需要覆盖哪些特征的人群？要多少人才算全面？

**找人的关键原则：代表性样本（Representative Sample）**
- 别只找你最容易找到的人。
- 刻意去触达过去研究里"被忽视的声音"——不同年龄、种族、性别、身体能力的人，使用方式天差地别。

**实际找人路径（由近到远）**：
1. 熟人、同事、同学
2. 社交媒体、LinkedIn、兴趣社群（如 Reddit、Facebook 小组）
3. 第三方招募机构 / 付费平台（UserTesting、UserInterviews——职场实战用）

**邀约邮件必备要素**：
- 自我介绍 + 项目背景
- 时间、地点（线上/线下）、时长
- 访谈形式 + 录音/录像授权说明
- 确认参与的方式
- **激励措施**（礼品卡最能提升参与意愿）
- ⚠️ 提前一周 + 前一晚各发一次提醒邮件，爽约率大幅降低

### 2. 访谈前准备

- **写好脚本**：问题列表保持一致，每个用户问同一套，方便后续对比
- **备齐物资**：问题清单、电脑/纸笔、录音设备
- **提前测试技术**：线上面试先熟悉平台，确保对方也能用；准备 Plan B（比如对方电话）
- **研究受访者**：提前看对方的基本信息、筛选问卷答案，开场就能聊到点上
- **自己先练一遍**：把问题读出声，别扭的地方现场会放大十倍

### 3. 正式访谈四步法

| 步骤 | 核心动作 | 实用技巧 |
|---|---|---|
| **① 破冰** | 让用户放松 | 闲聊开场；感谢对方时间；说明隐私和录音授权；强调"**没有标准答案**" |
| **② 提问** | 挖真实想法 | **只问开放式问题**——用 Why / What 开头。❌ "你喜欢网购吗？" ✅ "网购时你最爽和最烦的分别是什么？" |
| **③ 记录** | 捕捉信息 | 原话引用（加引号！）、表情/肢体/情绪变化、**用户做了什么有时比说了什么更重要** |
| **④ 收尾** | 完美闭环 | 问"还有什么想补充的？"→ 最后往往能炸出黄金洞察；真诚感谢；发放激励 |

**录音建议**：新手强烈建议录，事后可以反复听。但必须事先征求同意。

## 三、同理心地图（Empathy Map）：把混沌访谈变成可视化洞察

以用户为中心，画一个"十字"，分四象限：

| | |
|---|---|
| **说 Says**<br>用户的原话引用 | **想 Thinks**<br>没明说但可从表情、犹豫、语气推断的内心戏 |
| **做 Does**<br>用户的行为、操作步骤 | **感 Feels**<br>情绪：焦虑？兴奋？挫败？ |

**两种用法**：
- **单人版**：把一个典型用户的访谈精华浓缩成一张图
- **聚合版**：把同一类用户群体的反馈合并，提炼共性。注意：**不同细分群体要分别建图**

> 💡 **工作启发**：做产品/服务改进时，拉上团队一起填一张同理心地图，比念十页调研报告管用得多。

## 四、痛点（Pain Points）：设计师的靶心

UX 设计师的终极身份是 **Problem-Solver（问题解决者）**。痛点分四大类：

| 类型 | 典型场景 | 例子 |
|---|---|---|
| **财务痛点** | 钱的问题 | 读到一半突然跳出付费墙 |
| **产品痛点** | 质量/体验缺陷 | "诺曼门"——该推的门做成拉的 |
| **流程痛点** | 从 A 到 B 被卡住 | 购物车到结账步骤繁琐，用户流失 |
| **支持痛点** | 需要帮助却找不到 | 网站上客服电话藏得比宝藏还深 |

**进阶洞察**：用户有时连自己需要什么都不知道。
- 翻盖机时代，大家说要更长续航、更大按键，没人说要智能手机。
- 乔布斯的厉害之处，是**创造了用户尚未意识到的需求**。

> 所以做研究不只是"问用户要什么"，更是**观察用户的行为裂隙**——他们在哪里皱眉、叹气、放弃？

## 五、用户画像（Persona）：让"用户"从数据变成活人

基于真实研究，虚构一个具体人物来代表一类用户群体。

**画像里放什么**：
- 照片 + 基本档案（年龄、职业、居住地、家庭状况）
- 职业目标与日常职责
- 技术熟练度
- 一句口头禅（Catchphrase）

**为什么花力气做画像？**

1. **建立同理心**：
   - 说"全球有 53.3 万百岁老人"，你无感。
   - 说"Mavis Hunter，100 岁才开始跑步，现在是一名竞技跑者"，你立刻想为她做点什么。
   - **数据让人知道，故事让人在乎。**

2. **讲故事说服利益相关者**：
   - 拿着"Tsering Choedon，内布拉斯加乡村非营利组织创始人"的画像去开会，比放一堆统计表更能推动决策。

3. **压力测试设计**：
   - 适合 Mavis（百岁跑者）的功能，未必适合 Diane（带三个娃的职场妈妈）。
   - 多个人像互相制衡，防止设计只服务"最吵的用户"。

> ⚠️ **关键原则**：一个产品需要**一组画像**，每个核心用户群体都得有一个代表。只做一个是偷懒，做多了是负担，3-5 个通常最实用。

---

## 六、本模块速查清单（可直接抄到工作笔记）

- [ ] 访谈前：目标清晰 → 样本多元 → 脚本一致 → 设备就绪
- [ ] 访谈中：开放式问题 → 追深挖细 → 记录原话 + 观察
- [ ] 访谈后：填同理心地图 → 归类痛点 → 提炼画像
- [ ] 设计时：用人像做决策参照 → 问自己"Diane 会怎么做？Mavis 能理解吗？"

---

> **最后一句心法**：UX 的同理心不是"我为你好"，而是"我就是你"。当你能准确说出用户脑子里那一句没说出口的"唉，好烦"时，你就对了。
