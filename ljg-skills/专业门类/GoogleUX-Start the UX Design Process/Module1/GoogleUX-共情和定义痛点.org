* 问题

做产品的人，最大的幻觉是什么？是觉得自己知道用户要什么。

你刷了十年外卖App，你以为自己懂外卖用户。但你的痛点是"优惠券找不到"，另一个人的痛点可能是"字太小，我有阅读障碍"。UX设计的第一步不是画草图，是杀死这个幻觉——你真的理解坐在屏幕对面那个人吗？

这堂课讲的就是：怎么把自己清空，去听用户说话。

* 翻译

** 锚点：访谈就像相亲

想象你去相亲。你不会一上来就问"你存款多少"，也不会只盯着对方说了什么、不看表情。你会观察、会追问、会把偏见咽回去。访谈用户一模一样的道理。整个Module1就是教你怎样做好这场"相亲"，从约人、见面、聊天到事后整理。

** 放下自己：同理心六招

课程先给了六个动作，帮你练出"用户视角"：

| 动作 | 大白话 |
|------|--------|
| 多问 | 别猜，直接问"你怎么做的""为什么" |
| 多看 | 不只听 words，看表情、看动作、看环境 |
| 多听 | 全神听，别想下一句怎么回 |
| 多请人 | 别只问朋友，朋友会说好话的，找陌生人 |
| 放空 | 偏见人人都有，先认出来，再关进笼子 |
| 追新知 | 心理学在变，用户也在变，别拿五年前的套路套今天的人 |

同情（sympathy）和同理心（empathy）差在哪？同情是"我替你难过"，同理心是"我感受你的难过"。设计要的是后者。

** 访谈不是聊天：三件事定生死

一场好的访谈，前期定七成。课程用"Zia's Pizza"外卖App当例子，把准备工作拆成三块：

1. *目标*——你想知道什么？
   例子："我想懂忙人怎么安排吃饭的""我想懂点餐时哪里让人抓狂"

2. *找人*——谁值得你花时间？
   例子：18-62岁、住城里、每周至少点一次外卖、性别/能力要多样

3. *问题*——怎么问才能挖出真东西？
   五个原则：相关、开放、短、中立、能追问。
   烂问题："你点餐是因为没时间做还是懒得做？"——这是诱导。
   好问题："你最近一次点餐是什么时候？当时怎么想的？"

** 怎么把人约来

筛选用问卷（screener survey），问年龄、地点、职业、性别——但要把敏感问题设为选填，并说明"我们在做包容性设计，想确保接触到多样人群"。

找人渠道像漏斗：
- 熟人 → 现成用户池 → 社交媒体/专业平台（UserTesting、User Interviews） → 线下" hallway testing "（咖啡馆抓人） → 花钱请第三方 agency

越往后越贵，但越能触到真实多样的人群。

** 访谈四步走

第一步，见面破冰。闲聊天气、感谢对方来、说明没有标准答案、收集基础信息。让对方放松。

第二步，正式聊。语速放慢、保持专业、问开放题。如果对方只回"是/不是"，追问"能多说一点吗"。

第三步，狂记。记原话（引号括起来）、记表情动作、记环境干扰。有条件就录音，但必须先问许可。

第四步，收尾。给机会补充——"还有什么想说的吗？"——再谢一次。很多人在这时候才掏真心话。

** 访谈前的"预演"

课程给了一个聪明的做法：在见真人之前，先拿虚构用户画像（user profile）练手。

比如有个虚构用户Reena：37岁急救员、独居、有青少年女儿、轮班制、有阅读障碍、关注健康和预算。设计师在访谈前，先把自己当成Reena，想象她会怎么回答每一个问题。结果发现问题3"你遇到什么挑战？"可能会让有障碍的用户感到被冒犯或自我暴露。于是改成"你的点餐体验是怎样的？"——等到用户自己提到困难，再追问。

这叫 pre-interview empathy exercise。它的好处不是"假装懂用户"，而是提前暴露自己问题问得有多蠢。

** 把混沌变清晰：同理心地图

聊完一堆人，信息是乱的。同理心地图（empathy map）就是一张四格表，把一个人的访谈拆成：

+-----------+-----------+
|  Says     |  Thinks   |
| （原话）   | （推测想法）|
+-----------+-----------+
|  Does     |  Feels    |
| （行为）   | （情绪）   |
+-----------+-----------+

以遛狗App的Makayla为例：
- Says: "我不知道怎么判断谁来我家是安全的"
- Does: 早上晚上自己遛、付钱请邻居17岁儿子代遛
- Thinks: 想提前筛查遛狗人、想固定预约
- Feels: 愧疚（陪狗时间少）、担心（狗生病怎么办）

单人的地图攒多了，可以合成 aggregate empathy map，看出一群人的共同模式。

** 让数字变人：用户画像

同理心地图帮你拆，用户画像（persona）帮你合。把采访中反复出现的主题捏成一个虚构的人，让团队讨论时有个具体对象。

课程举了筹款App的例子：研究发现乡村单身专业人士给环保组织捐钱的频率是城里人的两倍，但城里人更愿意当志愿者。于是捏出一个 persona "Tsering"——35岁、内布拉斯加、环保组织创始人、不太懂技术、口头禅"gettin' greener every day"。

为什么要捏人？因为"53.3万百岁老人"你无感，但"Mavis Hunter，100岁才开始跑步"你忘不了。persona 就是给团队一个能感受的人，而不是一串报表。

** 找痛点：用户卡在四类地方

用户遇到的问题，课程分成四类：

| 类型 | 例子 |
|------|------|
| 钱（Financial） | 读到一半弹 paywall |
| 产品（Product） | Norman Door——门看着该推，结果要拉 |
| 流程（Process） | 购物车到结账绕了八步 |
| 支持（Support） | 客服电话藏得比宝藏还深 |

找痛点的工具链就是上面这些：访谈 → 同理心地图 → persona → 归类痛点。

* 核心概念

** 同理心（Empathy）

一句话：把自己放进用户的鞋子里，感受他的感受和想法。

类比：玩密室逃脱。同情是站在外面看监控说"他们好惨"；同理心是你在里面一起找钥匙、一起着急。

为什么重要？因为设计师太容易替用户做决定。没有同理心，你解的是自己假设的问题，用户真正的痛点被完美错过。

** 同理心地图（Empathy Map）

一句话：把一次访谈拆成"说、想、做、感"四格，逼你从四个角度看同一个人。

类比：看一部电影。Says 是对话字幕，Thinks 是内心独白，Does 是动作场面，Feels 是背景音乐。单看字幕你会漏掉一半故事。

为什么重要？因为人会说谎、会省略、会不自知。Thinks 和 Feels 填的是 Says 和 Does 之间的裂缝。没有这个工具，设计师只会听到用户嘴上说的，猜不到心里想的。

** 用户画像（Persona）

一句话：把一堆用户的共同特征捏成一个虚构的具体人物，让团队做决策时有 face 可对。

类比：足球队教练不看"平均球员数据"排兵，他想"梅西今天状态怎么样"。Persona 就是那个"梅西"——虽然虚构，但让讨论从抽象变具体。

为什么重要？没有 persona，团队开会时说"用户可能不喜欢"，谁都能插嘴。有了 persona，争论变成"Tsering 会怎么用？"——有靶子了。

* 洞见

真正值钱的不是"要访谈"或"要做 empathy map"这些动作。而是这个意识：用户的痛点往往和他嘴里说的不一样——你需要同时读他的心、读他的行为、读他的情绪，再把它们拼成一张图，才能看到真相。

* 博导审稿

- *选题眼光*：问题抓得准。"设计师容易自嗨"是行业老毛病，课程对症下药，对零基础非常友好。
- *方法成熟度*：流程完整。从筛人、约人、破冰、提问、记录到整理工具链，一环扣一环。缺点是偏入门，对复杂产品（B2B、医疗、金融）的访谈特殊性几乎没提。
- *实验诚意*：这不是论文，没有实验。但案例给得实在——Zia's Pizza、Reena、Makayla、Tsering，都是能帮助理解的 good enough 例子。
- *写作功力*：视频+阅读材料互补，视频讲感觉，阅读材料给 checklist。但两者有重复，阅读材料可以更精炼。
- *判决*：weak accept。作为入门课无可挑剔，但学过一遍后需要找更深入的实战资料补，不然会停留在"知道怎么做"而"不敢真做"。

* 启发

- *迁移*：课程里的"访谈前 empathy exercise"可以搬到任何需要理解客户的场景——写文案前先虚构一个目标读者、做销售前先想象客户的一天。具体做法：列五个问题，把自己当成那个人写答案，再看哪里问得蠢。
- *混搭*：同理心地图的四格（Says/Thinks/Does/Feels）可以和团队回顾会混用。项目复盘时，不用"哪里做得好哪里不好"，而是"当时我们说了什么、心里其实怎么想、做了什么、感受如何"——更容易挖出真问题。
- *反转*：课程假设"访谈越多越好"，但现实中有时用户自己也不知道自己要什么。乔布斯不做用户访谈。提醒自己：访谈解决的是"已知需求的优化"，突破性创新可能来自观察而非询问——该停下"只问不做"，开始"边看边试"。
